# Qt 6.8 Linux Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Linux deb packages with Qt 6.8 bundled so Ubuntu 24.04 users are not dependent on system Qt 6.4.

**Architecture:** GitHub Actions installs Qt 6.8 with `jurplel/install-qt-action`, builds the application against that Qt, then uses `linuxdeployqt` to collect Qt runtime files into a staging directory. The existing deb packaging flow packages the staged application under `/opt/qteasytier` and starts it through a wrapper that prefers bundled Qt libraries, plugins, and QML modules.

**Tech Stack:** CMake, Qt 6.8, GitHub Actions, linuxdeployqt, Debian packaging scripts, Bash.

## Global Constraints

- Build-time Qt version floor is 6.8.
- The generated deb must run on Ubuntu 24.04 without relying on Ubuntu's Qt 6.4 packages.
- The installed desktop entry must launch a wrapper, not `appQtEasyTier` directly.
- The daemon service remains installed as `qtet-daemon.service` and still executes `/opt/qteasytier/qtet-daemon`.
- CI may download `linuxdeployqt` only because the user explicitly approved using it.
